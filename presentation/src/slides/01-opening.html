Lean: From Functional Programming to Theorem Proving

One typed core language keeps more of a program's meaning visible at compile time.

Guiding question

What more can Lean know statically about a program?

The whole deck follows that one question from ordinary types to dependent types to proofs.

Roadmap

One talk, three stages

01

Functional programming

Expressions, definitions, inductive data, recursion.

02

Dependent types

Types can remember relationships that ordinary types forget.

03

Proofs as programs

A theorem becomes a type, and a proof is evidence inhabiting it.

Not three separate topics. One idea revealing itself in three stages.

What Is Lean?

"

One typed core language for programs, types, and proofs.

Lean is interesting because the second and third items are not bolted onto the first one.

Lean in practice

  • Functional programming language
  • Interactive theorem prover
  • Used for mathematics, software verification, and teaching
  • Best understood as a PL story, not a product tour

The key move is that the type system can carry much richer meaning.
